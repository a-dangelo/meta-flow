Workflow: it_ticket_submission
Description: Submit IT support tickets with intelligent routing based on issue type
Access: employee
Category: it

Inputs:
- issue_type (string): Type of IT issue (hardware, software, network, access, other)
- priority (string): Priority level (low, medium, high, critical)
- description (string): Detailed description of the issue
- device_id (string): Device identifier or hostname
- employee_id (string): Employee submitting ticket
- location (string): Physical location of employee or device

Steps:
1. Validate issue description is not empty
2. Check if device_id exists in asset management system
3. Search knowledge base for similar resolved issues
4. Create new IT ticket with issue details and assigned priority
5. ORCHESTRATOR WORKFLOW (switch on issue_type):
   - hardware → field support team
   - software → application development team
   - network → infrastructure team
   - access → security team
   - other/default → general IT support queue
   (Use an orchestrator node, not conditionals.)
6. Assign ticket to next available agent from routed team
7. Send confirmation email to employee with ticket number
8. Send notification to assigned support agent
9. Update IT dashboard metrics with new ticket

Outputs:
- ticket_id (string): Unique ticket identifier
- assigned_agent (string): Name of assigned support agent
- assigned_team (string): Team handling the ticket
- estimated_resolution_time (string): Expected resolution timeframe
